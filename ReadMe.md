# LMS Деятель

## Описание проекта
Следует реализовать LMS-систему, которая будет включать в себя следующий функционал:
1. Возможность создание дисциплин и подготовку к ним занятий:
    1. Материалы к занятию: видео-лекция, конспект занятия, книги, полезные ссылки (поля строго разделить, требовать .md к конспектам занятий); 
    2. Вид занятия:
        1. Традиционная модель: лекция, семинар, лабораторная.
        2. Модель А. А. Вербицкого: 
            1. Информационная (традиционная) лекция:
                - Появляются поле "тема", "проблемы/вопросы";
                - Обязательным становятся поля для материалов к лекции;
                - Напоминание составителю: преподаватель должен быть готов к дискуссиям;
                - Напоминание составителю: аудитория должна быть в виде круглого стола / амфитеатра
            2. Лекция вдвоем:
                - Обязательны материалы к занятию ("наглядные материалы", "дидактический материал" двух преподавателей);
                - Обязательное поле "сценарий лекции";
                - Обязательное поле "приглашённый лектор";
                - Обязательна репетиция такого занятия: необходимо оповещение приглашённого преподавателя, добавление в его расписание.
            3. Лекция-визуализация (презентация):
                - Обязательный материалы к занятию (презентации, файлы .pdf);
                - Обязательные поля "тема", "проблема/вопросы";
                - Необходимо отметить: важна подготовка визуальных представлений предполагаемого материала.
                - Необходимо напомнить: подача изображения должна быть такой, чтобы студент сам мог её воспроизвести из логических соображений.
            4. Лекция с запланированными ошибками;
                - Требования те же, что и к традиционной лекции
                - Появляется поле "ошибки по ходу лекции";
                - Появляется скрытое пространство для файлов "с ошибками";
                - Напоминание составителю: основные материалы не должны содержать ошибок.
            5. Лекция пресс-конференция:
                - Напоминание составителю: этот тип лекции создан для ответов на вопросы студентов.
            8. Деловая игра:
                - Обязательное поле "тема";
                - В соответствие с А. А. Вербицким "Структура деловой игры".
    3. Формат проведения:
        1. Дистанционное занятие: прикладывается ссылка
        2. Очное занятие: указывается набор подходящих аудиторий.
    4. Создание резервных занятий; 
    5. Формирование занятий для среза знаний: контрольная точка, самостоятельная работа, зачёт/экзамен.
    6. Определить формат сдачи (преподаватель оценивает каждого студента):
        1. Ничего не сдавать
        2. Развёрнутые ответы на вопросы
        3. Прикрепить файл 
    7. Уровень погружения в дисциплину: начальный (не профильный) / продвинутый (профильный) / полноценный (научный, принципиально новый курс в программе)


2. Возможность формирование направлений, каждому направлению - курсы, как набора дисциплин.
3. Возможность доступа студентов к материалам курса к которому их добавили.
4. Возможность обратной связи внутри системы студентов с преподавателем:
    1. Личные комментарии к каждому проведённому занятию;
    2. Публичные комментарии (относительно однокурсников).

5. Формирование расписания каждой группе студентов:
    1. Возможность учёта времени работы преподавателя;
    2. Ручное составление расписания (от лица преподавателя);
    3. Учёт расписания как группы, так и преподавателя.

6. Возможность простого экспорта расписания в вид для импорта расписания в любой календарь (.ics формат);
7. Формирования полноценных методических материалов на основе конспектов лекций (соединение всех лекций воедино + оглавление и титульная страница);
8. Ведение и просмотр статистики: 
    1. "География студента": теплокарта по субъектам РФ, откуда поступил студент;
    2. Успеваемость каждого студента во время курса, средняя успеваемость по группе;
    3. Посещаемость каждого студента, по группе, по всему ВУЗу.

9. Создание пространства для ведения конспектов занятий:
    1. Создание пространства для прочтения .md файлов (в формате Obsidian)
    2. Создание пространства для редактирования (с импортированием, экспортированием) .md файлов (в рамках одного единственного файла)

## Проблемы

1. Отсутствие систем, удовлетворяющих как традиционной модели организации, так и модели контекстного обучения.
2. Отсутствие таких LMS–систем, которые избегали бы дублирования информации от курса к курсу.
3. Отсутствие организации современной и удобной LMS-системы с возможностью ведения записей для создания материалов в учебной дисциплине.
4. Часто отсутствует возможность удобного эскпорта, синхронизации данных с календарём цифровым.
5. Монополия Moodle, Google Classroom (как иностранных решений).
6. Большой человеческий фактор по отношению к доступу к материалу дисциплины (выдуманный пример: Василий Петрович забыл передать перечень литературы, ибо посчитал, что он уже доступен).
7. Проблема актуальности расписания: существуют ситуации, когда для изменения расписания необходимо гигантское согласования и обращения в другие отделы для изменения расписания по факту. Формально составленное расписание есть, но со временем оно меняется в пользу частных улучшений.  

## Актуальность

1. Монополия Moodle, Google Classroom (не подходят по предлагаемому формату)
2. Полуавтоматическое расписание: перенос занятия можно сделать с учётом фактического расписания и группы, и преподавателя.
3. Организация удобного пространства для создания и ведения записей по каждому предмету, единый источник информации, уменьшение человеческого фактора "забыл скинуть" / "лень скинуть". Решение вопроса недоступности материала. 
4. Автоматизация сбора статистики: и преподаватель сможет видеть её, и студент сможет понять, что не так.
5. Существующие LMS не предоставляют систематизированной поддержки разных педагогических моделей (и традиционной, и контекстной), это вынуждает использовать внешние инструменты или не использовать их вовсе.


## Ролевая модель внутри проекта

1. Администратор: полный доступ;
2. Начальник направления: есть доступ к конечному набору направлений, к дисциплинам внутри;
3. Преподаватель: есть доступ к дисциплинам, полная возможность редактирования;
4. Староста: может оставить заявку на перенос занятия в другой день;
5. Студент: просмотр своего расписания, группы, каждого занятия.
